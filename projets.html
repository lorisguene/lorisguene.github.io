Gestion de fichiers

Le projet consiste à créer un script Python qui transforme un fichier JSON contenant les événements parisiens en un fichier CSV propre. Le script extrait et formate des champs spécifiques, convertit les dates/heures, respecte l’encodage, et assure la lisibilité du fichier sous Excel.

Technologies utilisées

  • Python : pour le traitement des fichiers JSON et la conversion en CSV
  • Pandas : pour la manipulation et le nettoyage de données structurées

Fonctionnalités clés

  • Extraction de champs spécifiques depuis un fichier JSON
  • Conversion des formats date/heure
  • Production de fichiers CSV propres et lisibles
  • Respect des encodages et compatibilité Excel

Durée du projet : 4 semaines

Conception et implémentation d’une base de données

Il s’agit d’un projet de modélisation (MCD, MLD), création de base sous MySQL, insertion de données, création de requêtes d’aide à la décision et réalisation d’une interface Tkinter. La base gère les apports des sauniers, les traitements, les ventes, et les prix annuels. L’application finale permet de visualiser, interroger et alimenter cette base.

Technologies utilisées

  • MySQL : base de données relationnelle pour stocker les informations
  • Python : pour l’interface graphique et l’interaction avec la base
  • Tkinter : pour créer l’interface utilisateur
  • Looping : pour la modélisation du MCD/MLD

Fonctionnalités clés

  • Création de la base avec MCD/MLD/MOT
  • Interface pour afficher et interroger les données
  • Requêtes SQL complexes d’aide à la décision
  • Ajout et modification d’enregistrements via GUI

Création d’un reporting

Ce projet a pour objectif de développer un outil Excel personnel de gestion des notes en BUT SD. Il permet l’ajout/modification de notes, l’affichage de tableaux de bord interactifs avec graphiques, et fournit un verdict automatique (admission, compétences validées) selon des règles précises. Un guide d’utilisation accompagne l’outil.

Technologies utilisées

  • Microsoft Excel : pour les tableaux, visualisations et mises en forme conditionnelles
  • VBA (Visual Basic for Applications) : utilisé ponctuellement pour automatiser certaines tâches complexes (formulaires, filtrages, navigation)

Fonctionnalités clés

  • Tableau de bord automatisé pour suivre les notes
  • Formules conditionnelles pour l’évaluation
  • Graphiques dynamiques liés aux données
  • Interface conviviale pour la saisie

Régression linéaire sur des données réelles

Ce projet repose sur un jeu de données réelles de ventes immobilières. Il faut choisir les variables pertinentes, construire un modèle de régression en R sans fonctions toutes faites (lm, predict), expliquer la démarche, fournir un code clair et rendre un fichier de prédictions à partir d’un jeu test. L’objectif est la précision du modèle, évaluée par la somme des carrés des résidus.

Technologies utilisées

  • R : pour le développement du modèle statistique
  • RStudio : environnement de développement
  • Excel : pour la préparation initiale des données

Fonctionnalités clés

  • Construction manuelle d’un modèle de régression
  • Choix et nettoyage des variables explicatives
  • Évaluation de la précision par les résidus
  • Génération d’un fichier de prédictions

Durée du projet : 2 semaines

Echantillonnage et Estimation

Le projet vise à estimer le nombre d’habitants d’une région à partir d’un échantillon, d’abord aléatoire, puis stratifié. L’étudiant analyse les marges d’erreur, construit des intervalles de confiance et compare les méthodes d’échantillonnage. Une deuxième partie s’intéresse aux relations entre variables qualitatives dans une enquête sportive étudiante.

Technologies utilisées

  • R : pour les méthodes d’échantillonnage et d’estimation
  • RStudio : pour le traitement statistique

Fonctionnalités clés

  • Échantillonnage aléatoire et stratifié
  • Calcul d’intervalles de confiance
  • Analyse croisée de variables qualitatives
  • Interprétation de résultats statistiques

Durée du projet : 1 semaines

Construction et présentation d’indicateurs de performances

Ce projet propose une étude de cas autour de Fleury-Michon à partir de données réelles (chiffres d’affaires, résultats, indicateurs). L’étudiant calcule des taux de variation, ratios, graphiques et construit un tableau de bord visuel. L’objectif est de diagnostiquer la santé financière d’une filiale, en la comparant à des ratios sectoriels.

Technologies utilisées

  • Excel : pour le calcul des indicateurs
  • Canva : pour la mise en forme du tableau de bord

Fonctionnalités clés

  • Calcul de ratios économiques
  • Comparaison avec les moyennes sectorielles
  • Création de graphiques explicites
  • Présentation visuelle synthétique
